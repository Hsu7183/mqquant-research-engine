# SYSTEM_SPEC

## 專案定位

本 repository 為 **mqquant 策略研究引擎**，用途是支援策略研究、驗證、淘汰與決策治理。

本專案核心目標不是追求「回測第一名」，而是：

> **淘汰假冠軍，保留可持續、可驗證、可交接的策略候選。**

## 系統角色分工

### ChatGPT（策略大腦與審查者）

- 負責策略邏輯推演、研究方向審查、驗證設計質疑。
- 負責檢查是否有資料洩漏、過度擬合、評估偏誤。
- 針對候選策略提出「可否升級 baseline」之審查意見。

### Codex（工程執行者）

- 負責將規格轉為可執行文件、流程、程式與可重複產出。
- 依既定規範建立任務輸出、測試紀錄與交接文件。
- 不擅自擴充規格邊界，遇到模糊需求先收斂為文件定義。

### GitHub（共同事實來源）

- 所有規格、任務、審查結論、版本變更皆應落地於 GitHub。
- 以版本控制維持「單一真實來源（Single Source of Truth）」。
- 所有人員與代理（人類 / ChatGPT / Codex）皆以 repo 內容為準。

## 主線版本

目前主線為：

- `01-01-01`

所有後續規格、驗證與交接，均應以此主線為基準進行比對與延伸。

## 目前階段定位

目前屬於第一階段：**文件與規格建置期**。

本階段重點：

1. 釐清交易與回測核心前提。
2. 釐清 OOS / WFO 驗證邏輯與輸出欄位。
3. 建立可交接、可審查、可追溯的文件骨架。

本階段非重點：

- 不先建立程式模組。
- 不先導入大量歷史資料。
- 不先追求最佳化績效數值。
