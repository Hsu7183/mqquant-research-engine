## 目前已完成

### 1. 單頁策略研究開發工作台

- 左側負責資料路徑、搜尋模式、執行設定、硬性過濾。
- 右側負責說明、執行監控、最佳化結果、驗證區驗收、全期間比較、KPI、逐筆交易。
- 首頁有連續式使用說明，不再使用分頁說明。
- 說明區最上方有四步驟主線：設定、最佳化、驗收、總結。

### 2. 執行中畫面已重排

- 按下開始後，如果背景任務正在跑，右側會隱藏使用說明。
- 執行中時右側最上方優先顯示 `即時執行監控`。
- `檢視背景程序監控`、進度、目前最佳候選會接在監控區下方。
- 停止與下載操作列會出現在監控區下方，不再擋在說明區後面。

### 2-1. 開發區年份可選

- 左側 `執行設定` 新增 `開發區年份`，可選 1 ~ 5 年。
- 1 年代表只用 2020，2 年代表 2020~2021，以此類推。
- 驗證區是剩餘資料區間：1 年開發 / 約 5 年驗證，2 年開發 / 約 4 年驗證，以此類推。
- 圖表橘色分隔線是驗證區起點，不再固定為最近 1 年起點。
- 不同開發區年份會使用不同 `research_profile_tag`，避免歷史 Top10 / incumbent 混用不同開發區結果。

### 2-2. MDD 門檻可做批次範圍測試

- 左側 `硬性過濾` 新增 `MDD 門檻模式`。
- `固定門檻` 維持原本單一 `最大 MDD(%)`。
- `範圍測試` 可輸入 `MDD 起點(%)`、`MDD 終點(%)`、`MDD 步長(%)`，例如 3% ~ 15%。
- MDD 仍是硬過濾：範圍測試時會以範圍上限作為淘汰門檻，並在同一批回測結果中分層統計各 MDD 門檻的候選表現。
- 執行中與執行後會顯示 `MDD 門檻批次結果`，不用一個 MDD 門檻按一次開始。

### 2-3. 多輪 WFO / gap 骨架

- 左側 `執行設定` 新增 WFO 設定：
  - `WFO 訓練窗(年)`
  - `WFO 測試窗(年)`
  - `WFO 滑動步長(年)`
  - `WFO gap 天數`
- 單次最佳化完成後，右側新增 `多輪 WFO 驗證` 區塊。
- 按 `計算多輪 WFO` 後，每一輪會：
  - 依訓練窗重新跑最佳化。
  - 訓練窗與測試窗之間先空出 gap。
  - 保留該輪前 N 名候選。
  - 將候選丟到下一段測試窗做 OOS 驗收。
  - 輸出 OOS 報酬、OOS 最大回撤、OOS 交易數、OOS Sharpe 與是否通過最低門檻。
- 總表會顯示總輪數、通過輪數、失敗輪數、平均 OOS 報酬、平均 OOS MDD、最差一輪報酬、最差一輪 MDD、OOS 穩定度與通過率。

### 2-4. 多層策略驗證與決策系統

- 結果頁新增 `多層策略驗證與決策系統`。
- 目前會明確列出：
  - 第 1 層：單次 OOS 初篩。
  - 第 2 層：多輪 WFO。
  - 第 3 層：gap / purge 防污染，現階段先做 train/test gap。
  - 第 4 層：PBO / DSR 近似版。
  - 第 5 層：Forward Test 前測。
  - 2 / 3 / 4 / 5 點滑價壓力測試。
  - 穩定區 / 尖峰警示。
- 最終建議會落在四種之一：
  - 維持 Baseline
  - 讓 Challenger 進前測
  - 讓 Challenger 升格
  - 暫停上線，繼續研究
- 已新增 `bundle/run_history/forward_test_log.csv`，可由 UI 寫入 Challenger 前測紀錄。

### 3. 最新最佳策略會自動回填到路徑

- 每次最佳結果落盤時，會另外產生：
  - `bundle\strategy\<命名>.xs`
  - `bundle\param_presets\<命名>.txt`
- `XS 路徑` 和 `參數範圍 preset` 會自動切到最新產生的兩個檔案。
- 命名規則為：`民國年3碼 + 月日 + 時分 + 總報酬率整數`。
- 範例：2026-04-21 08:20，總報酬率 133%，檔名為 `11504210820133`。

### 4. Worker 控制與穩定性

- 背景程序有 heartbeat / stale 判斷。
- 瀏覽器關掉後，不應讓 Python worker 無限殘留。
- 背景程序監控提供手動停止與清理。
- 驗證時不要用長掛式 `streamlit run ...` 卡住流程。
